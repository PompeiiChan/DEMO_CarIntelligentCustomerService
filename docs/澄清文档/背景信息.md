1. 智能客服系统解决的核心问题
  客户背景：上汽大众售后客服体系
  核心痛点：
  1. 人工客服压力大：大量重复性、标准化咨询（车辆操作、功能使用、故障排查步骤等）占用人工时间
  2. 传统客服机器人智障：关键词匹配+规则树，理解能力差，用户只能转人工
  3. 服务成本高：每一通人工电话成本约10-20元，7×24小时覆盖需要大量人力
  解决目标：
  - 用AI接管60-70%的高频、标准化售后咨询（用车、保养、故障排查）
  - 通过多轮对话+上下文管理收集完整信息，减少"人工来回问"
  - 精确转人工：只在用户明确要求或遇到复杂场景时转人工，避免"AI答不上来就转人"的懒惰策略
  落地渠道：web

目标用户：
  主要用户：
  1. 潜在购车客户 - 售前咨询（车型、配置、价格、优惠）
  2. 已订车客户（售中） - 订单履约查询（生产排期、运输进度、交付时间）
  3. 已购车车主（售后） - 用车、保养、故障排查
  次要用户：客服人员（售前+售后）
  三大场景模块：
  - 售前：车型对比、配置咨询、价格政策、试驾预约
  - 售中：订单状态查询、生产进度、物流跟踪、交付安排
  - 售后：用车指导、保养提醒、故障排查、保修政策

用户会在什么场景下使用？
用户使用场景

  售前场景：
  - 在APP或企业微信咨询车型配置、价格对比、优惠活动
  - 询问贷款方案、试驾预约

  售中场景：
  - 已订车后，查询订单状态、生产排期进度
  - 追踪车辆运输物流、确认交付时间

  售后场景：
  - 用车过程中咨询功能操作（车机设置、功能使用）
  - 查询保养预约、维修价格、道路救援、保险理赔

  共同特点：7×24小时，随时随地，无需等待人工客服。

按道理来说应该是在手机上使用，但是我们主要开发Demo，所以我们做一个Web端，方便让他们拉起一下整个产品的链路。

这个系统大致需要包含哪些功能？
一、核心功能模块

  1. 对话状态管理

  - 5种状态机：空闲等待、人工服务中、人工解决后、新话题触发、延续对话
  - 介入判断：AI何时介入、何时退让、何时保持静默
  - 转人工策略：
    - 用户主动要求（"转人工""人工客服"）
    - AI连续3次未匹配答案
    - 用户情绪负面（"太垃圾了""没用"）
    - 涉及复杂维修/事故/投诉
    - 上下文传递：将用户问题、AI尝试、转人工原因传给人工客服

  2. 意图识别（CICC框架）

  - 意图分类：10+种汽车行业意图（车型咨询/价格咨询/订单查询/生产进度/保养预约/维修咨询/道路救援/保险理赔等）
  - 置信度判断：
    - 高置信度（预测集=1个）→ 直接回复
    - 中置信度（预测集=2-3个）→ 追问澄清（"请问您是需要道路救援吗？"）
    - 低置信度（预测集>5个）→ 引导用户重新描述
  - 意图纠错：用户说错时主动澄清（"车换了"→追问是换车还是换配件）

  3. 回复子流程（五路分流）

  售前Agent：
  - 车型对比推荐
  - 价格/优惠查询
  - 贷款方案计算
  - 试驾预约
  - 引导留资/到店

  售中Agent：
  - 订单状态查询（已下单/已排产/生产中/已下线/运输中/已到店）
  - 生产排期进度（预计第XX周完成）
  - 物流跟踪（当前到达XX物流中心）
  - 交付时间确认（预计XX日可提车）
  - 交付准备提醒（需准备身份证/购车款等）

  售后Agent：
  - 保养预约（查询保养记录/预约保养时间）
  - 维修咨询（故障排查指引/维修价格）
  - 道路救援（触发救援流程/告知救援范围）
  - 保险理赔（理赔流程指引/所需材料）
  - 保修政策查询

  车辆使用Agent：
  - 功能操作指引（车机设置/配置调节）
  - 警告灯解读（报警灯含义/应对措施）
  - 常见问题解答（油耗异常/异响等）

  兜底流程：
  - 未匹配意图时引导用户补充信息
  - 建议转人工或联系400热线

  4. 知识检索与管理

  - 三路知识源：
    - 车型知识（参数/配置/价格/活动）
    - 订单/生产/物流数据（实时接口查询）
    - 保养/维修/政策文档
  - RAG检索：向量检索+关键词混合检索
  - 知识优先级：结构化数据 > 文档 > 图片提取
  - 历史QA提取：从企业微信历史对话中沉淀高频问答

  二、系统能力

  5. 上下文管理

  - 多轮对话记忆：记住最近3-5轮对话，保持上下文连贯
  - 用户身份识别：关联用户订单/车辆信息
  - 渠道信息：区分APP/企业微信，适配回复风格

  6. 情绪识别

  - 正面/负面分类：识别用户不满情绪
  - 情绪响应：负面情绪时自动转人工或安抚回复

  7. 人工协作

  - 人机切换：AI↔人工无缝切换
  - 上下文传递：AI将对话摘要传给人工客服
  - 托管回收：人工处理后可将对话交回AI

  三、管理与优化

  8. 评测体系

  - 6级评分：
    - 完全正确且有帮助
    - 基本正确但需补充
    - 部分正确但有错误
    - 回答无关
    - 拒绝回答（应该能答但没答）
    - 幻觉/胡编
  - 数据飞轮：Bad Case回流→优化Prompt/知识库→再评测

  9. 迭代版本管理

  - V1 MVP：售前场景，Workflow Demo
  - V2 工程化：售前+售中+售后，模型分层，多Agent系统
  - V3 全场景：CICC框架、知识扩充、Bad Case飞轮

  10. 监控与统计

  - 核心指标：
    - 意图识别准确率（目标~85%）
    - AI承接率（目标~80%）
    - 单次对话成本（降低60%+）
    - 响应时间（1-2秒）
  - 实时监控：对话量、转人工率、情绪分布
